# OKF Harness 网站路线图

[English](../../website/ROADMAP.md) | 中文

本路线图面向首个可信首页发布后的 `okf-harness.dev` 公开网站。首个首页在本路线图之外跟踪；本文件记录后续网站方向，这些方向应等到该入口证明自身价值后再推进。

## 版本 2：真实产品证明

目标：让首页更具辨识度，同时不把它变成完整的网站系统。

- 在有稳定素材后，添加真实的终端截图、简短的工作流录屏或紧凑的演示图片。
- 仅在形成可复用的品牌视觉后，添加真实的社交分享预览图。
- 仅在项目拥有有意设计的标识后，添加真实的 favicon 或 logo。
- 保持安装部分与 README 和发布流程同步。

## 版本 3：文档导航

目标：改善导航，同时避免重复仓库文档。

- 仅当通过 GitHub README 导航成为可衡量的支持问题时，才添加轻量的文档入口页面。
- 除非镜像内容能给读者带来明确收益，否则继续将详细的 CLI、工作流、路线图和发布信息保留在仓库文档中。
- 在生成式文档网站确有必要之前，优先重定向到规范的仓库文档，而不是复制页面。

## 版本 4：本地化

目标：仅在英文首页稳定后添加本地化的网站入口。

- 在英文内容、布局和安装流程经过验证后，添加中文首页。
- 在发布多个语言版本前，引入 locale 生成、hreflang、sitemap 和漂移检查。
- 保持本地化安装命令和受支持智能体声明与 README 和发布资产同步。

## 后续可能方向

- 当形成值得在浏览器中展示的稳定首次使用故事后，添加简短的交互式演示。
- 当仅靠发布说明已无法清楚说明产品方向时，添加轻量的发布亮点。
- 仅当仓库文档对非开发者用户而言难以浏览时，才添加生成式文档网站。
- 仅当存在需要回答的具体问题且隐私边界已有文档说明时，才添加结构化的网站分析。

## 非目标

- 不构建独立的知识库应用。
- 不承诺云端同步、账户或托管 workspace。
- 除非产品模式发生变化，否则不添加定价页面。
- 不使用图库图片或通用产品截图。
