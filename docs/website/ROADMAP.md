# OKF Harness Website Roadmap

English | [中文](../zh-CN/website/ROADMAP.md)

This roadmap is for the public website surface at `okf-harness.dev` after the first trusted homepage ships. The first homepage is tracked outside this roadmap; this file records later website directions that should wait until that entry point has proven useful.

## Version 2: Real Product Proof

Goal: make the homepage more recognizable without turning it into a full site system.

- Add a real terminal screenshot, short workflow capture, or compact demo image when a stable asset exists.
- Add a real social preview image only after a reusable brand treatment exists.
- Add a real favicon or logo only after the project has an intentional mark.
- Keep the install section synchronized with the README and release process.

## Version 3: Documentation Routing

Goal: improve navigation without duplicating the repository docs.

- Add a lightweight docs landing route only if GitHub README routing becomes a measurable support problem.
- Keep detailed CLI, workflow, roadmap, and release information in the repository docs unless there is a concrete reader benefit to mirroring it.
- Prefer redirects to canonical repository docs over copied pages until a generated docs site is justified.

## Version 4: Localization

Goal: add localized website entry points only after the English homepage has stabilized.

- Add a Chinese homepage when the English content, layout, and install flow are proven.
- Introduce locale generation, hreflang, sitemap, and drift checks before shipping multiple locales.
- Keep localized install commands and supported-agent claims synchronized with the README and release assets.

## Later Possibilities

- Add a short interactive walkthrough once there is a stable first-use story worth showing in the browser.
- Add lightweight release highlights when release notes alone no longer explain the product direction well.
- Add a generated docs site only if repository docs become hard for non-developer users to navigate.
- Add structured website analytics only if there is a concrete question to answer and the privacy boundary is documented.

## Non-Goals

- No standalone knowledge-base application.
- No cloud sync, account, or hosted workspace promise.
- No pricing page unless the product model changes.
- No stock imagery or generic product screenshots.
